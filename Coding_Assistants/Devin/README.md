# Devin — Prompt Guide

## Overview
Devin is an autonomous AI software engineer that can plan, implement, debug, and deploy code independently. These prompts are designed for delegating complete engineering tasks.

## How to Use

### System Prompt
1. Copy `System_Prompt.txt` content into Devin's system instructions or session preamble.
2. This establishes Devin's working methodology and quality expectations.

### User Prompts
- Use prompts as **task descriptions** (tickets) assigned to Devin.
- Be specific about requirements, constraints, and acceptance criteria.
- Include links to relevant documentation, files, and repositories.

## Tips for Best Results
- **Write clear acceptance criteria** — Devin works best with well-defined goals.
- **Provide context** — Link to existing code, docs, and related issues.
- **Set constraints** — Specify technologies, patterns, and approaches to use (or avoid).
- **Review incrementally** — Check Devin's progress at each phase rather than only at the end.
- **Give feedback** — If a solution isn't quite right, provide specific, actionable feedback.
